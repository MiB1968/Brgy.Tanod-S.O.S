## 2025-02-12 - Optimize high-frequency location updates
**Learning:** Real-time location tracking can overwhelm databases when handled via REST fallback calls (`gps/heartbeat`) and socket broadcast spam. A fully WebSocket-based approach with client-side throttling and server-side batching significantly reduces latency and server load.
**Action:** Removed REST `gps/heartbeat` API calls from `gpsService.ts`. Implemented a 2000ms client-side throttle for `tanod_move` socket events. Added server-side batching in `location.handler.ts` to queue and broadcast `location_update_batch` every 1000ms, updating the client to support the batched event.
